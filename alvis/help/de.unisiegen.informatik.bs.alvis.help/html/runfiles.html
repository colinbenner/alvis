Run Dateien

Run Dateien sind dazu da, um zu steuern, wie ein Algorithmus auf ein Beispiel angewandt wird.

Dazu wählt im Feld "Algorithm File" einen Algorithmus aus dem aktuellen Alvis-Workspace aus. Außerdem fügt man im Feld "Example File" ein Beispiel hinzu, das als Datum für den Algorithmus dient. Dies kann ein Graph, Syncronisationsproblem oder ein anderer beliebiger Datentyp aus Alvis-Plugins sein.

Symbol/Reiter

Bedeutung
Springe in den Editor der jeweiligen Datei
Starte den Ablauf mit den gegebenen Parametern
Start Points Zufällig: Alvis entscheidet, welche Parameter dem Algorithmus übergeben werden. Beispiel: Start- und Endknoten im Graph-Plugin
Entscheide: Der Benutzer entscheidet selbst. Sobald man den Run startet, wird man dann nach den Parametern gefragt
Decision Points Jedes Mal, wenn der Algorithmus auf eine unsortierte Menge von Objekten (Beispiel: die Nachbarn eines Knoten in einem Graph) stößt, muss eine Entscheidung getroffen werden, in welcher Reihenfolge diese Daten abgearbeitet werden.
Zufällig: Alvis entscheidet zufällig
Einmal: Der Benutzer legt einmal eine Reihenfolge fest, die auch beim Zurückgehen im Algorithmus nicht gelöscht wird
Immer: Jedes Mal wird vom Nutzer eine Entscheidung verlangt. Frühere Entscheidungen werden beim Zurückgehen im Algorithmus gelöscht

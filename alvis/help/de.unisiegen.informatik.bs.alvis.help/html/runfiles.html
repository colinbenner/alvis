Run Files

Run Files are a means of controlling how an algorithm is used on an example.

In order to do so you chose an algorithm file from the current Alvis workspace. Furthermore you add an example file on which the algorithm is used. This can be a graph or a syncronisation problem oder any other datatype provided by an Alvis plug-in.

Symbol/Tab

Description
Open the editor of the particular file.
Start the run with the given parameters.
Start Points Random:Alvis decides, which parameters are handed over to the algorithm. Example: Start and end node of a graph algorithm.
Decide:The user decides. As soon as the run starts, Alvis asks for the parameters.
Decision Points Every time the algorithm has to process an unordered set of data (example: adjacent nodes of node in a graph), the user has to decide, in what order the data is used.
Random: Alvis decides randomly.
Once:The user orders the data belonging to each decision points once. These decisions will not be delete when undoing algorithm steps.
Always: Alvis asks for a decision every time and decisions will be deleted by going backwards.
